# 先验、后验、似然：调参即贝叶斯推断
{: .fs-6 }

{: .note }
本文不追求数学严格，目标是让「贝叶斯推断」成为你重新理解 LLM 调参的一把直觉工具。可先阅读 [LLM 输出为什么是概率分布（待写）] 建立直觉，再读本文。

## 一个日常比喻

你第一次用某个 LLM，问它「今天天气怎么样」，它回答「可能是晴天，但也可能下雨」。

这不是模型在「思考」——而是它**在你提问之前，就已经有一个关于世界的主观判断**（先验）。你的提问（evidence）更新了它的判断，得到「更确信是晴天」（后验）。

这个过程，就是贝叶斯推断。

---

## 三个词各自的直白含义

### 先验 P(θ) — 模型在看见数据之前「相信」什么

在 LLM 语境下，先验是**模型架构本身对世界的预设**：

- 你的 prompt 是「你是一个严谨的数据分析师」——这是在加一个先验，让模型偏好保守输出
- fine-tuning 时用特定行业数据训练——也是在加先验：让模型相信这个领域的信息更可靠
- temperature = 0.2——强制模型选最高概率token——这是一种「强先验」，抑制模型探索不确定的答案

{: .highlight }
prompt engineering 本质上是在**改变模型的先验**，而不是在「改变模型」。

### 似然 P(D|θ) — 给定这个信念，现实给出什么反馈

当你把一批标注数据（问题+答案）喂给模型，模型要问自己：**在这些数据面前，我原来的信念有多合理？**

fine-tuning 的过程就是在计算这个：给定当前模型参数 θ，数据 D 出现的概率有多大？

- 如果数据 D 和先验 θ 一致 → 似然高 → 后验也会倾向于保持 θ
- 如果数据 D 严重偏离 θ → 似然低 → 后验会大幅修正 θ

在 RLHF（人类反馈强化学习）里，人类对回答的打分就是提供「似然信号」：高分回答提升了「这个回答是好的」这个假设的似然。

### 后验 P(θ|D) — 更新之后，模型「现在」相信什么

贝叶斯定理把三者串起来：

```
后验 ∝ 似然 × 先验

P(θ|D) ∝ P(D|θ) × P(θ)
```

模型在看见数据之后更新的那套信念，就是后验。

fine-tuning 之后模型的权重变了，是因为它在数据 D 的基础上，学会了「修正」原来的先验。

## 调参场景中的贝叶斯直觉

### 场景一：prompt 里加「冷静、严谨」

你加了一句 system prompt：「回答要基于已知事实，不要编造」。

**贝叶斯视角：** 你在把模型的先验从「倾向说完整的话」（高熵先验）拉向「只说有把握的话」（低熵先验）。模型的输出分布变得更集中，幻觉概率下降。

### 场景二：调整 temperature

| temperature | 先验效果 | 典型场景 |
|-------------|---------|---------|
| 0（几乎 deterministic） | 强先验：只信最确定的 token | 代码生成、精确抽取 |
| 0.7（中等探索） | 弱先验：愿意考虑多个候选 | 创意写作、头脑风暴 |
| 1.2（高随机） | 几乎无先验：高度不确定 | 探索性对话、打破思维定势 |

{: .important }
temperature 是一种**不需要重新训练模型就能改变先验**的操作。在 toB 场景里，针对不同任务切 temperature，比换模型更实用。

### 场景三：fine-tuning vs RAG

| 方案 | 改变了什么 | 先验变化 |
|------|-----------|---------|
| fine-tuning | 模型权重 θ | 把行业知识编码进先验 P(θ) |
| RAG（检索增强） | 推理时输入 D | 在似然计算时注入新证据 |
| prompt engineering | system prompt | 加一个即时先验，推理时叠加 |

RAG 之所以有效，是因为它**不改变模型的先验，而是改变了模型看见的证据**——模型在推理时把新证据纳入似然计算，得到更贴近真实分布的后验。

{: .highlight }
如果你发现 fine-tuning 后模型反而「学坏了」（原有领域知识被稀释），那就是**新数据的似然压过了原来的先验**，导致后验偏离了目标。这时 RAG 或 prompt 注入往往比继续 fine-tune 更稳。

---

## 写在最后

把调参想成贝叶斯推断，可以帮助你：

- **不是盲目试参数**，而是有方向地理解每一次调参在影响贝叶斯链条的哪一环
- 在 toB 场景里，更有根据地决定：这个问题是用 prompt、fine-tune 还是 RAG
- 向非技术背景的客户解释「为什么模型这次答错了」——贝叶斯语言比梯度下降更直觉
